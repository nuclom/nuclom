## Priority Definitions

### P1 - Critical

**Definition**: Complete service outage affecting business operations

**Examples**:
- Cannot access Nuclom at all
- All video playback failing
- Authentication completely broken
- Data loss or security incident

**Escalation**: Immediate engineering engagement

### P2 - High

**Definition**: Major feature completely unavailable

**Examples**:
- Video uploads failing for all users
- Transcription service down
- Cannot add team members
- API returning errors consistently

**Escalation**: Senior support engagement within response time

### P3 - Medium

**Definition**: Feature partially working with degraded experience

**Examples**:
- Slow video load times
- Delayed transcription (>30 min for short videos)
- Search results incomplete
- Notifications delayed

**Escalation**: Normal support queue with priority handling

### P4 - Low

**Definition**: Non-urgent issues and general questions

**Examples**:
- How to use a specific feature
- Feature enhancement requests
- Billing questions
- Documentation clarification

**Escalation**: Standard support queue

## How to Submit a Support Request

### Required Information

For fastest resolution, include:

1. **Organization Name**: Your Nuclom organization
2. **Affected Users**: How many users are impacted
3. **Issue Description**: Clear description of the problem
4. **Steps to Reproduce**: How to recreate the issue
5. **Expected vs Actual**: What should happen vs what is happening
6. **Screenshots/Videos**: Visual evidence of the issue
7. **Browser/Device**: Technical environment details

### Email Format Example

```
Subject: [P2] Video uploads failing for all team members

Organization: Acme Corp
Affected Users: All 15 team members

Issue Description:
When attempting to upload any video file, the upload starts but fails
at 50% with error "Upload failed - please try again."

Steps to Reproduce:
1. Go to My Videos
2. Click "New Video"
3. Select any video file (tested MP4, MOV)
4. Upload progress reaches ~50% then fails

Expected: Video should upload completely
Actual: Upload fails at approximately 50%

This started happening today at 2pm EST.

Screenshots attached.
Browser: Chrome 120 on macOS
```

## Escalation Process

### Pro Plan Escalation

1. **Initial Response**: Support agent acknowledges issue
2. **Investigation**: Agent investigates and provides updates
3. **Engineering Escalation**: If unresolved after 2 updates, escalates to engineering
4. **Resolution**: Fix deployed and verified

### Enterprise Plan Escalation

1. **Immediate Triage**: Dedicated support reviews immediately
2. **Account Manager Notification**: Your AM is informed of P1/P2 issues
3. **Engineering Fast-Track**: Direct access to engineering team
4. **Executive Escalation**: For extended outages, executive involvement
5. **Post-Incident Review**: Detailed analysis shared after resolution

## Support Best Practices

### Do's

- Provide detailed information upfront
- Include screenshots and error messages
- Specify the urgency level accurately
- Respond promptly to follow-up questions
- Test suggested solutions thoroughly

### Don'ts

- Don't mark non-critical issues as P1
- Don't submit duplicate tickets
- Don't include sensitive data in screenshots
- Don't wait to report critical issues
