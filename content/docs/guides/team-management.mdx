## Invite Members

<Steps>
  <Step title="Open Members settings">
    Go to **Organization Settings** → **Members**.
  </Step>
  <Step title="Send invitation">
    Enter the teammate’s email and select a role.
  </Step>
  <Step title="Track invitations">
    Pending invites appear in the Members list so you can resend or revoke them.
  </Step>
</Steps>

## Roles and Permissions

Nuclom uses role-based access to control what members can do.

<CardGroup cols={2}>
  <Card title="Default Roles" icon="user">
    Organizations start with default roles like **Owner** and **Member**.
  </Card>
  <Card title="Custom Roles" icon="shield">
    Create custom roles with granular permissions for videos, collections, and settings.
  </Card>
</CardGroup>

### Create or Edit Roles

<Steps>
  <Step title="Open Roles settings">
    Go to **Organization Settings** → **Roles**.
  </Step>
  <Step title="Add or edit">
    Create a new role or update an existing one.
  </Step>
  <Step title="Configure permissions">
    Choose the resources and actions each role can access.
  </Step>
</Steps>

## Manage Members

| Action | Where to do it |
| ------ | -------------- |
| Change role | **Organization Settings** → **Members** |
| Remove access | **Organization Settings** → **Members** |
| Review permissions | **Organization Settings** → **Roles** |

## Security Best Practices

<AccordionGroup>
  <Accordion title="Enable SSO">
    Configure SAML or OIDC in **Organization Settings** → **SSO** for centralized access control.
  </Accordion>
  <Accordion title="Review audit logs">
    Track sign-ins and admin actions in **Organization Settings** → **Audit Logs**.
  </Accordion>
  <Accordion title="Limit admin access">
    Keep admin roles small and use custom roles for specialized workflows.
  </Accordion>
</AccordionGroup>
